# Processed Embeddings Directory

This directory contains precomputed token embeddings generated from the raw TinyStories dataset. Embeddings are stored as chunked PyTorch .pt files to optimize memory usage and I/O performance during training.

## File Structure and Naming Convention
Files follow the pattern: embeddings_{model}_{split}_chunk_{nnnn}.pt
Examples:
- embeddings_bert-mini_train_chunk_0000.pt
- embeddings_distilbert_val_chunk_0001.pt

## Supported Model Filters
- bert-mini (prajjwal1/bert-mini, 256d)
- distilbert (distilbert-base-uncased, 768d)
- minilm (all-MiniLM-L6-v2, 384d)
- tinybert (TinyBERT_General_4L_312D, 312d)
- jina (jina-embeddings-v2-small-en, 512d)

## Technical Details
Format: PyTorch .pt files saved in float16 precision.
Content: Each chunk contains batched sequences of shape (B, T, d_in), where B is batch size, T is sequence length (up to 512), and d_in is the hidden dimension of the base encoder.
Storage: Float16 storage reduces disk footprint and I/O bandwidth by 50 percent. Values are automatically upcast to float32 when loaded by the DataLoader.
Chunking: Data is split into manageable chunks to prevent RAM overload during dataset preloading.

## Generation and Usage
Generated by: scripts/02_gen_embeddings_*.py
Consumed by: src/dataset.py and all training/analysis scripts in src/
Regeneration: Safe to delete individual chunks or the entire directory. They can be regenerated from data/raw/train.txt and data/raw/val.txt using the generation scripts.

## Note
Do not manually edit or modify these files. They are intermediate artifacts intended solely for model training and evaluation.
